# Layout

Layout is a page-shell component that composes `AlertBanner`, `TopBar`, `Sidebar`, and a main content region into a responsive layout, typically using a `Grid` for the main content.

It provides consistent structure across tools while allowing each region to be populated with existing Stand components.

## When to use

- When your page needs a standard shell and layout with a top bar and main content grid.
- When you need optional global messaging via `AlertBanner`.
- When you need secondary navigation or contextual content in a `Sidebar`.
- When you want mobile-specific sidebar behavior (`hidden` or `above-grid`) without building custom layout logic.

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`
- `react-aria-components` (for the `TopBar` components used in the example)

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Example usage

<SandboxReact componentName={componentName} componentTsx={componentTsx} />

```tsx
import { Grid, Item } from '@guardian/stand/grid';
import { Layout, Sidebar } from '@guardian/stand/layout';
import { TopBar, TopBarToolName } from '@guardian/stand/TopBar';

<Layout>
	<TopBar>
		<TopBarToolName name="Layout Demo" favicon={{ letter: 'L' }} />
	</TopBar>
	<Sidebar layoutSmBreakpoint="above-grid">
		<div style={{ padding: '8px' }}>Sidebar content</div>
	</Sidebar>
	<Grid>
		<Item size={{ sm: 12, md: 8 }}>Main content</Item>
		<Item size={{ sm: 12, md: 4 }}>Secondary content</Item>
	</Grid>
</Layout>;
```

## Composition

`Layout` inspects its direct `children` and places the first child of each supported type into the corresponding layout region:

| Region       | Accepted child type | Grid area name |
| ------------ | ------------------- | -------------- |
| Alert banner | `AlertBanner`       | `alertbanner`  |
| Top section  | `TopBar`            | `topbar`       |
| Sidebar      | `Sidebar`           | `sidebar`      |
| Main content | `Grid`              | `main`         |

Other direct children are still rendered, but `Layout` does not assign them to a grid area or inject layout styles.

If you want a custom child to occupy a specific region without using one of the supported components, you must manually set `grid-area` on that child to one of the layout area names above.

`Layout` only inspects direct children; nested children or those wrapped in a `React.Fragment` are not supported.

This means you can compose the shell declaratively, and `Layout` will place each supported child in the correct grid area.

## Layout Props

| Name           | Type                                     | Required | Default | Description                                                                                     |
| -------------- | ---------------------------------------- | -------- | ------- | ----------------------------------------------------------------------------------------------- |
| `children`     | `React.ReactNode`                        | No       | N/A     | Direct children used to populate alert banner, top bar, sidebar, and any custom layout content. |
| `fluid`        | `boolean`                                | No       | `true`  | When `false`, constrains the main region to the theme max width and centers it.                 |
| `theme`        | `LayoutTheme`                            | No       | N/A     | Theme token overrides for layout grid structure across breakpoints.                             |
| `cssOverrides` | `SerializedStyles \| SerializedStyles[]` | No       | N/A     | Custom CSS styles.                                                                              |
| `className`    | `string`                                 | No       | N/A     | Additional class name(s).                                                                       |

## Sidebar Props

Use `Sidebar` as a child of `Layout` to fill the sidebar region.

| Name                 | Type                                     | Required | Default    | Description                               |
| -------------------- | ---------------------------------------- | -------- | ---------- | ----------------------------------------- |
| `children`           | `React.ReactNode`                        | No       | N/A        | Content to render inside the sidebar.     |
| `layoutSmBreakpoint` | `'above-grid' \| 'hidden'`               | No       | `'hidden'` | Sidebar behavior at the `sm` breakpoint.  |
| `theme`              | `SidebarTheme`                           | No       | N/A        | Theme token overrides for sidebar styles. |
| `cssOverrides`       | `SerializedStyles \| SerializedStyles[]` | No       | N/A        | Custom CSS styles.                        |
| `className`          | `string`                                 | No       | N/A        | Additional class name(s).                 |

### `layoutSmBreakpoint` behavior

- `hidden`: Sidebar is hidden on `sm` and appears again from `md` upward. This is shown in `TopBarGridSidebarHiddenSm`.
- `above-grid`: Sidebar is shown on `sm` and appears above the main grid content, then returns to the side from `md` upward. This is shown in `TopBarGridSidebarAboveGridSm`.

## Customisation

We recommend using the Layout component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the Layout component (for example grid templates at each breakpoint, and shared width constraints used when `fluid={false}`):

<Canvas of={CustomTheme} />

```tsx
import type { LayoutTheme } from '@guardian/stand/layout';
import { Layout } from '@guardian/stand/layout';

const customTheme: Partial<LayoutTheme> = {
	sm: {
		gridTemplateAreas: `'topbar' 'sidebar' 'main'`,
		gridTemplateColumns: '1fr',
		gridTemplateRows: 'auto auto 1fr',
	},
};

const Component = () => <Layout theme={customTheme}>{...}</Layout>;
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the Layout component:

<Canvas of={CssOverrides} />

```tsx
import { Layout } from '@guardian/stand/layout';
import { css } from '@emotion/react';

const customStyles = css`
	border: 2px dashed #9aa0a6;
	border-radius: 8px;
	overflow: hidden;
`;

const Component = () => <Layout cssOverrides={customStyles}>{...}</Layout>;
```
