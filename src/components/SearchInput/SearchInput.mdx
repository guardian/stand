# SearchInput

A search input component built on [React Aria `SearchField`](https://react-aria.adobe.com/SearchField) component, see their documentation for the API.

It provides a labelled, accessible search field with support for descriptions, validation errors, two sizes, and a disabled state.

## When to use

- Within a form to collect a single line of free-text input from the user, or a specific type of input such as an email address/password (with appropriate `type` prop).
- Whenever an accessible, labelled search field is needed that aligns with Stand tokens

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `react-aria-components`
- `typescript`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

```tsx
import { SearchInput } from '@guardian/stand/SearchInput';

/* types, if required */
import type {
	SearchInputProps,
	SearchInputTheme,
} from '@guardian/stand/SearchInput';

const Example = () => (
	<>
		<SearchInput
			label="Name"
			description="This is a description for the search input."
		/>

		<SearchInput
			size="sm"
			label="Username"
			isInvalid
			error="Username is already taken."
			defaultValue="guardian_user"
		/>

		<SearchInput label="Notes" isDisabled placeholder="Read only content" />
	</>
);
```

| Name                      | Type                                       | Required | Default | Description                                                                                                                                     |
| ------------------------- | ------------------------------------------ | -------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `size`                    | `'md'` \| `'sm'`                           | No       | `'md'`  | Controls the input height and typography.                                                                                                       |
| `label`                   | `string`                                   | No       | N/A     | Visible label rendered above the input.                                                                                                         |

| `description`             | `string`                                   | No       | N/A     | Helper text rendered below the label.                                                                                                           |
| `error`                   | `string`                                   | No       | N/A     | Validation error message; displayed when `isInvalid` is `true`.                                                                                 |
| `isInvalid`               | `boolean`                                  | No       | `false` | Marks the field as invalid and applies error border styling.                                                                                    |
| `isDisabled`              | `boolean`                                  | No       | `false` | Disables interaction and applies disabled styling.                                                                                              |
| `theme`                   | `SearchInputTheme` (partial)               | No       | N/A     | Override Stand tokens for this instance; merged over the default theme.                                                                         |
| `placeholder`             | `string`                                   | No       | N/A     | Temporary text that occupies the search input when it is empty.                                                                                 |
| `formInputContainerTheme` | `FormInputContainerTheme` (partial)        | No       | N/A     | Override Stand tokens for the wrapping label and description container.                                                                         |
| `cssOverrides`            | `SerializedStyles` \| `SerializedStyles[]` | No       | N/A     | Low-level escape hatch for Emotion overrides when theming is insufficient.                                                                      |
| `className`               | `string`                                   | No       | N/A     | Optional class name forwarded to the root React Aria SearchField element.                                                                       |
| `...props`                | `ReactAria SearchField` props              | No       | N/A     | All other props from `react-aria-components` [`SearchField`](https://react-aria.adobe.com/SearchField) (e.g. `value`, `onChange`, `autoFocus`). |

## Customisation

We recommend using the SearchInput component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the SearchInput component:

```tsx
import type { SearchInputTheme } from '@guardian/stand/SearchInput';
import { SearchInput } from '@guardian/stand/SearchInput';
import { baseColors } from '@guardian/stand';

const customTheme: Partial<SearchInputTheme> = {
	shared: {
		border: `0.0625rem solid ${baseColors.coolPurple[500]}`,
		borderRadius: '0.5rem',
		error: {
			border: `0.0625rem solid ${baseColors.red[500]}`,
		},
	},
};

const Component = () => <SearchInput label="Email" theme={customTheme} />;
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the SearchInput component:

```tsx
import { SearchInput } from '@guardian/stand/SearchInput';
import { css } from '@emotion/react';

const customStyles = css\`
	font-style: italic;
\`;

const Component = () => (
	<SearchInput label="Notes" cssOverrides={customStyles} />
);
```
