# Grid

Grid is a flexbox layout primitive with a split API: use `Grid` as the outer layout container, and wrap each child in an `Item` to control its column span.

## When to use

- When you need a predictable multi-column layout with configurable gaps.
- When you need simple responsive sizing with the repo's `sm`, `md`, and `lg` breakpoints.

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Example usage

<SandboxReact componentName={componentName} componentTsx={componentTsx} />

```tsx
import { Grid, Item } from '@guardian/stand/Grid';

/* types, if required */
import type { GridProps, GridTheme, ItemProps } from '@guardian/stand/Grid';

<Grid>
	<Item size={{ sm: 12, md: 8, lg: 6 }}>Main content - sm:12, md:8, lg:6</Item>
	<Item size={{ sm: 12, md: 4, lg: 3 }} offset={{ md: 1, lg: 2 }}>
		More content - sm:12, md:4, lg:3, offset md:1, lg:2
	</Item>
	<Item size="auto">Auto width</Item>
	<Item size="grow">Grows to fill</Item>
</Grid>;

<Grid as="section" aria-label="Homepage highlights">
	<Item size={12}>Featured content</Item>
</Grid>;
```

By default, `Grid` renders a `div`. Use the `as` prop to render a semantic element when needed.

For accessibility landmarks, only use `as="main"` when it is the single primary landmark for that document section.

## Grid Props

| Name           | Type                                     | Required | Default | Description                                                       |
| -------------- | ---------------------------------------- | -------- | ------- | ----------------------------------------------------------------- |
| `children`     | `React.ReactNode`                        | No       | N/A     | Content to render inside the container.                           |
| `as`           | `React.ElementType`                      | No       | `'div'` | Element used to render the Grid container.                        |
| `theme`        | `GridTheme`                              | No       | N/A     | Theme tokens controlling columns, gap, and container flex layout. |
| `cssOverrides` | `SerializedStyles \| SerializedStyles[]` | No       | N/A     | Custom CSS styles.                                                |
| `className`    | `string`                                 | No       | N/A     | Additional class name(s).                                         |

Container layout values such as `direction`, `wrap`, `justifyContent`, and `alignItems` now come from the `theme` prop rather than direct component props.

### Grid Theme

The `GridTheme` type is structured with a `shared` key for flex layout properties and per-breakpoint keys (`sm`, `md`, `lg`) for column count, gap, and padding.

**`shared`**

| Name                    | Type              | Default        | Description                     |
| ----------------------- | ----------------- | -------------- | ------------------------------- |
| `shared.display`        | `string`          | `'flex'`       | Container display type          |
| `shared.width`          | `string`          | `'100%'`       | Container width                 |
| `shared.direction`      | `flex-direction`  | `'row'`        | Container flex direction        |
| `shared.wrap`           | `flex-wrap`       | `'wrap'`       | Container wrap behavior         |
| `shared.justifyContent` | `justify-content` | `'flex-start'` | Main-axis distribution of items |
| `shared.alignItems`     | `align-items`     | `'stretch'`    | Cross-axis alignment of items   |

**Per-breakpoint (`sm` / `md` / `lg`)**

| Name        | Type     | sm default | md default | lg default | Description                                      |
| ----------- | -------- | ---------- | ---------- | ---------- | ------------------------------------------------ |
| `*.columns` | `number` | `12`       | `12`       | `12`       | Total columns available to items                 |
| `*.gap`     | `string` | `'16px'`   | `'16px'`   | `'16px'`   | Gap between rows and columns                     |
| `*.padding` | `string` | `'16px'`   | `'24px'`   | `'24px'`   | Horizontal padding applied to the Grid container |

> **Note:** The `theme` prop accepts a deeply-partial override of this structure. For example:
>
> ```tsx
> theme={{
>   shared: {
>     direction: 'row',
>     wrap: 'wrap',
>     justifyContent: 'flex-start',
>     alignItems: 'stretch',
>   },
>   sm: { columns: 12, gap: '16px', padding: '16px' },
>   md: { columns: 12, gap: '16px', padding: '24px' },
>   lg: { columns: 12, gap: '16px', padding: '24px' },
> }}
> ```

## Customisation

We recommend using the Grid component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the Grid component:

<Canvas of={CustomTheme} />

```tsx
import type { GridTheme } from '@guardian/stand/Grid';
import { Grid, Item } from '@guardian/stand/Grid';

const customTheme: GridTheme = {
	shared: {
		justifyContent: 'space-between',
		alignItems: 'flex-start',
	},
	sm: { gap: '8px' },
	md: { gap: '16px' },
	lg: { gap: '16px' },
};

const Component = () => (
	<Grid theme={customTheme}>
		<Item size={6}>Column one</Item>
		<Item size={6}>Column two</Item>
	</Grid>
);
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the Grid component:

<Canvas of={CssOverrides} />

```tsx
import { Grid, Item } from '@guardian/stand/Grid';
import { css } from '@emotion/react';

const customStyles = css\`
	padding-block: 24px;
\`;

const Component = () => (
	<Grid cssOverrides={customStyles}>
		<Item size={8}>Primary content</Item>
		<Item size={4}>Sidebar</Item>
	</Grid>
);
```

## Custom Component Build

If you're not using React/Emotion, you can create a custom Grid component using the styles defined in the `GridTheme` type.

Since we're using flexbox to implement the grid, you'll need to understand how the offset and sizing works.

The calculations for the `size` and `offset` props are as follows:

- For `size`:
  - If a number is provided, the width is based on the available row width after gaps are accounted for. In other words, the implementation starts from the container width minus the total horizontal gaps, takes `(size / columns)` of that value, and then adds back the internal gaps spanned by those columns. With `columns` columns and horizontal `gap`, this is effectively `((100% - (columns - 1) * gap) / columns) * size + (size - 1) * gap`. When `gap` is `0`, this reduces to `(size / columns) * 100%`.
  - If `auto` is provided, the item will shrink to fit its content.
  - If `grow` is provided, the item will grow to fill available space after fixed-size items are laid out.
- For `offset`:
  - If a number is provided, the left margin is also gap-aware. The implementation takes `(offset / columns)` of the container width minus the total horizontal gaps, then adds back the gaps crossed by that offset. With `columns` columns and horizontal `gap`, this is effectively `((100% - (columns - 1) * gap) / columns) * offset + offset * gap`. When `gap` is `0`, this reduces to `(offset / columns) * 100%`.
  - If `auto` is provided, the item will be pushed to the rightmost edge of the container.
