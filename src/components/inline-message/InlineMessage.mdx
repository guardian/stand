# InlineMessage

A compact inline component used to display contextual feedback messages. It pairs an icon with a short text message and supports three severity levels: `error`, `success`, and `information`.

## When to use

- To give the user inline feedback following an action (e.g. form validation errors, submission success)
- To surface contextual status information close to the relevant UI element
- When a full alert or banner would be too visually prominent

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Props

| Name           | Type                                     | Required | Default | Description                                                                                                                               |
| -------------- | ---------------------------------------- | -------- | ------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `level`        | `'error' \| 'success' \| 'information'`  | Yes      | N/A     | The severity level of the message. Determines colour and default icon.                                                                    |
| `children`     | `React.ReactNode`                        | No       | N/A     | The message content to render inside the component.                                                                                       |
| `hideIcon`     | `boolean`                                | No       | `false` | When `true`, hides the default icon associated with the message level.                                                                    |
| `icon`         | `string \| SVGElement`                   | No       | N/A     | Overrides the default icon, for example if you're not using Material Symbols. Accepts a Material Symbols name (string) or an SVG element. |
| `theme`        | `Partial<InlineMessageTheme>`            | No       | N/A     | Custom theme overrides for design tokens.                                                                                                 |
| `cssOverrides` | `SerializedStyles \| SerializedStyles[]` | No       | N/A     | Custom Emotion CSS styles applied to the root element.                                                                                    |
| `className`    | `string`                                 | No       | N/A     | Additional class name(s) applied to the root element.                                                                                     |

## Customisation

We recommend using the InlineMessage component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the InlineMessage component. The available tokens mirror the `InlineMessageTheme` type:

<Story of={CustomTheme} />

```tsx
import { baseColors } from '@guardian/stand';
import type { InlineMessageTheme } from '@guardian/stand/inline-message';
import { InlineMessage } from '@guardian/stand/inline-message';

const customTheme: Partial<InlineMessageTheme> = {
	error: {
		color: baseColors.orange[400],
	},
};

const Component = () => (
	<InlineMessage level="error" theme={customTheme}>
		Something went wrong
	</InlineMessage>
);
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the InlineMessage component:

<Story of={CssOverrides} />

```tsx
import { css } from '@emotion/react';
import { baseColors } from '@guardian/stand';
import { InlineMessage } from '@guardian/stand/inline-message';

const customStyles = css`
	background-color: ${baseColors.green[50]};
	padding: 16px;
	border-radius: 4px;
`;

const Component = () => (
	<InlineMessage level="information" cssOverrides={customStyles}>
		Your session will expire soon
	</InlineMessage>
);
```
