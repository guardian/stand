# TopBar

A top bar that provides information about the tool, the current state and navigation links among other things.

## When to use

- At the top of a tool
- To identify the tool, its current state and the user
- To navigate around the tool

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Example usage

<SandboxReact componentName={componentName} componentTsx={componentTsx} />

```tsx
import {
	TopBar,
	TopBarContainerLeft,
	TopBarItem,
} from '@guardian/stand/TopBar';

/* types, if required */
import type { TopBarProps, TopBarTheme } from '@guardian/stand/TopBar';

<TopBar theme={customTheme}>
	<TopBarToolName name="Author" favicon={{ letter: 'A' }} />
	<TopBarContainerLeft>
		<TopBarItem>Hello</TopBarItem>
	</TopBarContainerLeft>
	<TopBarContainerRight>
		<TopBarNavigation text={'Link to somewhere'} href="#" />
	</TopBarContainerRight>
	<Avatar
		src="https://uploads.guimcode.co.uk/2026/01/27/f85e2e477ce54f4c3b671faa5cd21673aa9f8072fddb5d70a73e6038dc812eec.jpg"
		alt="Mahesh Makani"
		size="sm"
	/>
</TopBar>;
```

## Composition

The `TopBar` component is composed of several sub-components. Only children of type `TopBarToolName`, `Avatar`, `TopBarContainerLeft` or `TopBarContainerRight` are valid.

The children will be rendered in the following order from left to right:

| `TopBarToolName` | `TopBarContainerLeft` | `TopBarContainerRight` | `Avatar` |

`TopBarToolName` will always be on the left and `Avatar` on the right.

### Props

| Name            | Type                                                    | Required | Default                                   | Description                                                                                                                                                 |
| --------------- | ------------------------------------------------------- | -------- | ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `children`      | `React.ReactNode`                                       | No       | N/A                                       | Content to render inside the component. Only children of type `TopBarToolName`, `Avatar`, `TopBarContainerLeft` or `TopBarContainerRight` will be rendered. |
| `collapseBelow` | `{ toolName?: Breakpoint; containerLeft?: Breakpoint }` | No       | `{ toolName: 'lg', containerLeft: 'lg' }` | Controls at which breakpoint each section collapses. See [Responsive behaviour](#responsive-behaviour).                                                     |
| `theme`         | `TopBarTheme`                                           | No       | N/A                                       | Custom theme overrides.                                                                                                                                     |
| `cssOverrides`  | `SerializedStyles`                                      | No       | N/A                                       | Custom CSS styles.                                                                                                                                          |
| `className`     | `string`                                                | No       | N/A                                       | Additional class name(s).                                                                                                                                   |

### `TopBarContainerLeft` / `TopBarContainerRight` props

All other content must be composed within the left hand side or right hand side. The only valid children are [TopBarNavigation](/docs/stand-tools-design-system-components-topbar-topbarnavigation--docs) and [TopBarItem](/docs/stand-tools-design-system-components-topbar-topbaritem--docs).

`TopBarNavigation` children should be used for navigation links around the tool. `TopBarItem` provides the necessary styling for any other child components in the top bar.

| Name       | Type              | Required | Default | Description                                                                                                              |
| ---------- | ----------------- | -------- | ------- | ------------------------------------------------------------------------------------------------------------------------ |
| `children` | `React.ReactNode` | No       | N/A     | Content to render on one side of the top bar. Only children of type `TopBarItem` or `TopBarNavigation` will be rendered. |

## Responsive behaviour

The TopBar supports responsive design via the `collapseBelow` prop. This controls at which breakpoint each section of the top bar collapses.

Available breakpoints: `sm` (0px), `md` (672px), `lg` (1056px), `xl` (1312px), `max` (1584px), `maxplus` (1784px).

```tsx
import type { TopBarProps } from '@guardian/stand/TopBar';

// collapses at md and below (i.e. below 1056px)
<TopBar collapseBelow={{ containerLeft: 'lg', toolName: 'lg' }}>
```

### `containerLeft` collapse

When the viewport is below the specified breakpoint:

- The `TopBarContainerLeft` content is hidden.
- A **hamburger menu button** appears in its place.
- Clicking the button opens a popover containing the left container's `TopBarNavigation` and `TopBarItem` children.
- The menu closes automatically when the viewport is resized back above the breakpoint.

The default value for both `containerLeft` and `toolName` is `'lg'`, meaning both collapse below 1056px by default.

### `toolName` collapse

When the viewport is below the `toolName` breakpoint, the `TopBarToolName` displays in a shortened form. See the [TopBarToolName docs](/docs/stand-tools-design-system-components-topbar-topbartoolname--docs) for the `collapsedHoverText` prop that controls its collapsed label.

## Customisation

We recommend using the TopBar component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the TopBar component. This prop will also be propagated to its subcomponents that compose it.
This means for example to define an inverse colour scheme you only have to pass one prop.

```tsx
import type { TopBarTheme } from '@guardian/stand/TopBar';
import {
	TopBar,
	TopBarContainerLeft,
	TopBarItem,
} from '@guardian/stand/TopBar';
import { baseColors, semanticColors } from '@guardian/stand';

const customTheme: Partial<TopBarTheme> = {
	'background-color': baseColors.cyan[200],
	ToolName: {
		color: semanticColors.text['stronger-inverse'],
	},
	Navigation: {
		selected: {
			color: semanticColors.text['stronger-inverse'],
			'border-bottom': `${semanticSizing.border['extra-wide']} solid ${semanticColors.border['selected-inverse']}`,
		},
		unselected: {
			color: semanticColors.text['stronger-inverse'],
		},
	},
};

const Component = () => (
	<TopBar theme={customTheme}>
		<TopBarToolName
			name="Default"
			favicon={{
				letter: 'D',
				theme: {
					color: {
						background: baseColors.cyan[100],
						text: baseColors.cyan[900],
					},
				},
			}}
			href="#"
			hoverText="Back to dashboard"
		/>
		<TopBarContainerLeft>
			<TopBarNavigation isSelected text="Navigation 1" href="#" />
			<TopBarNavigation text="Navigation 2" href="#" />
			<TopBarNavigation text="Navigation 3" href="#" />
		</TopBarContainerLeft>
		<Avatar
			src="https://uploads.guimcode.co.uk/2026/01/27/f85e2e477ce54f4c3b671faa5cd21673aa9f8072fddb5d70a73e6038dc812eec.jpg"
			alt="Mahesh Makani"
			size="sm"
		/>
	</TopBar>
);
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the TopBar component:

```tsx
import { TopBar } from '@guardian/stand/TopBar';
import { baseColors } from '@guardian/stand';
import { css } from '@emotion/react';

const customStyles = css\`
	border: 4px solid ${baseColors.cyan['300']};
\`;

const Component = () =>
	<TopBar cssOverrides={customStyles}>
		<TopBarToolName name="CssOverrides" favicon={{ letter: 'C' }} />
		<TopBarContainerLeft>
			<TopBarItem>Hello</TopBarItem>
		</TopBarContainerLeft>
	</TopBar>
;
```
