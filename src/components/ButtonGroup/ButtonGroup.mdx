# ButtonGroup

The ButtonGroup component is used to lay out groups of buttons responsively with consistent spacing applied between them. By default buttons are positioned side by side with their content determining size. On mobile buttons are stacked vertically and displayed full width.

## When to use

- Presenting multiple related actions to the user (eg. within a modal or form)

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Example usage

<SandboxReact componentName={componentName} componentTsx={componentTsx} />

```tsx
import { ButtonGroup } from '@guardian/stand/ButtonGroup';
import type {
	ButtonGroupProps,
	ButtonGroupTheme,
} from '@guardian/stand/ButtonGroup';

const Example = () => (
	<ButtonGroup size="lg">
		<Button variant="tertiary">Stay on this step</Button>
		<Button>Discard changes and skip</Button>
	</ButtonGroup>
);
```

## Props

| Name           | Type                                 | Required | Default | Description                                    |
| -------------- | ------------------------------------ | -------- | ------- | ---------------------------------------------- |
| `size`         | `'xs'` \| `'sm'` \| `'md'` \| `'lg'` | No       | `'md'`  | Sets the size of the buttons within the group. |
| `theme`        | `ButtonGroupTheme`                   | No       | N/A     | Custom theme overrides for the ButtonGroup.    |
| `cssOverrides` | `SerializedStyles`                   | No       | N/A     | Custom CSS styles for the ButtonGroup.         |
| `className`    | `string`                             | No       | N/A     | Additional class name(s) for the ButtonGroup.  |

## Customisation

We recommend using the ButtonGroup component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the ButtonGroup component:

<Canvas layout="padded" of={CustomTheme} />

```tsx
import type { ButtonGroupTheme } from '@guardian/stand/ButtonGroup';
import { ButtonGroup } from '@guardian/stand/ButtonGroup';
import { semanticSpacing } from '@guardian/stand';

const customTheme: ButtonGroupTheme = {
	gap: semanticSpacing['stackXs'],
	justifyContent: 'flex-end',
};

const Component = () => (
	<ButtonGroup theme={customTheme}>
		<Button variant="tertiary">Stay on this step</Button>
		<Button>Discard changes and skip</Button>
	</ButtonGroup>
);
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the ButtonGroup component, if the theme prop is not sufficient:

<Canvas layout="padded" of={CssOverrides} />

```tsx
import { ButtonGroup } from '@guardian/stand/ButtonGroup';
import { baseColors, baseSizing, baseSpacing } from '@guardian/stand';
import { css } from '@emotion/react';

const customStyles = css`
	border: ${baseSizing.size2Rem} solid ${baseColors.red[500]};
	margin: ${baseSpacing['8Rem']};
	padding: ${baseSpacing['8Rem']};
`;

const Component = () => (
	<ButtonGroup cssOverrides={customStyles}>
		<Button variant="tertiary">Stay on this step</Button>
		<Button>Discard changes and skip</Button>
	</ButtonGroup>
);
```

## Custom Component Build

If you're not using React/Emotion, or `@guardian/stand` is not compatible with your project, you can create a custom ButtonGroup component using the styles defined in the `ButtonGroupTheme` type.

You will however be responsible for any additional functionality on top of the styles, for example image loading, image fallback, accessibility etc.

See "Custom Component" heading under the "ButtonGroup" component on [codesandbox.io](https://codesandbox.io/p/sandbox/guardian-stand-components-forked-j8cjhy)
