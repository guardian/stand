# Tooltip

Displays contextual help or information when the trigger element is hovered or focused.

## When to use

- To present additional information about a UI element or option
- To provide guidance or context without cluttering the interface

## Peer dependencies

- `@emotion/react`
- `react`
- `react-dom`
- `typescript`
- `react-aria-components`

See the `peerDependencies` section of `package.json` for compatible versions.

See [custom component build](#custom-component-build) for usage without React/Emotion.

## Example usage

<SandboxReact componentName={componentName} componentTsx={componentTsx} />

```tsx
import { Tooltip } from '@guardian/stand/Tooltip';
/* types, if required */
import type { TooltipProps, TooltipTheme } from '@guardian/stand/Tooltip';

const Example = () => <Tooltip placement="top">Tooltip text</Tooltip>;
```

## Props

| Name           | Type                                          | Required | Default | Description                               |
| -------------- | --------------------------------------------- | -------- | ------- | ----------------------------------------- |
| `placement`    | `'top'` \| `'bottom'` \| `'start'` \| `'end'` | No       | 'end'   | Position of tooltip relative to trigger.  |
| `theme`        | `TooltipTheme`                                | No       | N/A     | Custom theme overrides for the Tooltip.   |
| `cssOverrides` | `SerializedStyles`                            | No       | N/A     | Custom CSS styles for the Tooltip.        |
| `className`    | `string`                                      | No       | N/A     | Additional class name(s) for the Tooltip. |

## Customisation

We recommend using the Tooltip component as provided, but it can be customised using the `theme` or `cssOverrides` props as required.

### Custom theme

The `theme` prop allows you to override specific design tokens for the Tooltip component:

<Canvas layout="padded" of={CustomTheme} />

```tsx
import type { TooltipTheme } from '@guardian/stand/Tooltip';
import { Tooltip } from '@guardian/stand/Tooltip';
import { baseColors, semanticRadius } from '@guardian/stand';

const customTheme: TooltipTheme = {
	color: {
		shared: {
			color: baseColors.orange[900],
			backgroundColor: baseColors.orange[50],
		},
		tooltip: {
			borderRadius: semanticRadius.cornerMd,
		},
	},
};

const Component = () => <Tooltip theme={customTheme}>Tooltip text</Tooltip>;
```

### CSS overrides

The `cssOverrides` prop allows you to pass custom CSS to the Tooltip component, if the theme prop is not sufficient:

<Canvas layout="padded" of={CssOverrides} />

```tsx
import { Tooltip } from '@guardian/stand/Tooltip';
import { css } from '@emotion/react';

const customStyles = css`
	span {
		font-size: 1.25rem;
	}
	&[data-placement='right'] {
		margin-left: 1.5rem;
	}
	svg {
		width: 18px;
		height: 28px;
	}
`;

const Component = () => (
	<Tooltip cssOverrides={customStyles}>Tooltip text</Tooltip>
);
```

## Custom Component Build

If you're not using React/Emotion, or `@guardian/stand` is not compatible with your project, you can create a custom Tooltip component using the styles defined in the `TooltipTheme` type.

You will however be responsible for any additional functionality on top of the styles, for example image loading, image fallback, accessibility etc.

See "Custom Component" heading under the "Tooltip" component on [codesandbox.io](https://codesandbox.io/p/sandbox/guardian-stand-components-forked-j8cjhy)

**`css`**

You can import the Tooltip styles as CSS from the package, make sure that your build process supports importing CSS from `node_modules`/`package.json`:

<SandboxCss
	componentName={componentName}
	componentHtml={componentHtml}
	componentCss={componentCss}
/>

**TypeScript/JavaScript**

Use the `componentTooltip` variable and the `ComponentTooltip` type to define your custom styles in TypeScript/JavaScript:

<SandboxJs componentName={componentName} componentJs={componentJs} />
