# Semantic - Breakpoints

Semantic breakpoint tokens define the viewport width ranges used across the design system. They are the foundation for all responsive styling, and should be used via the media query API rather than referenced directly.

## Media query API

The recommended way to write responsive styles is with the `from`, `until`, and `between` helpers exported from `@guardian/stand/utils`. They return plain media query strings that work with any CSS-in-JS library or as object-style keys.

```ts
import { from, until, between } from '@guardian/stand/utils';
```

---

### `from`

Applies styles at the given breakpoint **and above** (`min-width`).

```ts
import { css } from '@emotion/react';
import { from } from '@guardian/stand/utils';

const styles = css`
	display: none;

	${from.md} {
		display: block;
	}
`;
```

You can also use object-style syntax (e.g. with Emotion's `css()` or a `style` prop):

```ts
const styles = css({
	display: 'none',
	[from.md]: {
		display: 'block',
	},
});
```

### `until`

Applies styles on viewports **below** the given breakpoint (exclusive `max-width`). The query ends just before the breakpoint starts, preventing overlap with `from`.

```ts
import { css } from '@emotion/react';
import { until } from '@guardian/stand/utils';

const styles = css`
	display: block;
	${until.md} {
		display: none;
	}
`;
```

### `between...and...`

Applies styles for viewports **within a range**, from breakpoint A up to (but not including) breakpoint B.

```ts
import { css } from '@emotion/react';
import { between } from '@guardian/stand/utils';

const styles = css`
	display: flex;
	${between.sm.and.lg} {
		flex-direction: column;
	}
`;
```

## Raw breakpoint tokens

If you need direct access to the pixel values, for example, to pass into a third-party component or to build a custom query, you can import the token object:

```ts
import { semanticBreakpoints } from '@guardian/stand';
import '@guardian/stand/semantic/breakpoints.css'; // CSS custom properties

// JS/TS
const minWidth = semanticBreakpoints.md.min; // '600px'

// CSS custom properties
// var(--semantic-breakpoints-md-min)  →  600px
// var(--semantic-breakpoints-md-max)  →  1023px
```

---

## When to use semantic breakpoints

- **`from`**
  - when a style should kick in at a given size and scale up from there (mobile-first).
- **`until`**
  - when something should only appear on smaller screens.
- **`between`**
  - for styles scoped to a specific viewport range, such as a tablet-only layout adjustment.
